# CH32H417飞控硬件原理图分析报告

> **串口分配最终确认（2026-05-31）**
> 本项目硬件分配最终以此为准：USART1=Debug(PB6/PB7)、USART2=接收机(PA3)、USART3=陀螺仪(PB10/PB11)、USART4=GPS(PC6/PC7)、USART5=PF5/PE0空闲、USART6=下视激光(PC10/PC11)、USART7=前视激光(PC12/PD2)、USART8=数传(PE8/PE7)、I2C3=罗盘(PA14=SCL/PA13=SDA)。

## 一、外设接口连接器分析

### 1. GPIO扩展接口
**连接器**: HDR-M_2.54_1x4P (4针排针)  
**引脚**: PC4, PC8, PC9, PD3, PD4  
**用途**: 通用GPIO扩展，可用于额外传感器或执行器

### 2. 陀螺仪/IMU接口
**连接器**: XH2.54-4P (4针座)  
**标注**: "陀螺仪"  
**引脚**: GND, PB10, PB11, 3V3  
**可能配置**:
- **USART3**: PB10=TX, PB11=RX (串口模式，460800波特率)
- **I2C2**: PB10=SCL, PB11=SDA (I2C模式)

**分析**: 根据项目文档，ATK IMU使用串口协议，因此应配置为USART3。

### 3. 下视激光传感器
**连接器**: GH1.25-4P (4针座)  
**标注**: "下视"  
**引脚**: 5V, GND, PC11, PC10  
**配置**: USART6_RX (PC11), 115200波特率  
**用途**: TFmini Plus下视测距，用于定高

### 4. 前视激光传感器
**连接器**: GH1.25-4P (4针座)  
**标注**: "前向"  
**引脚**: 5V, GND, PC12, PD2  
**配置**: USART7_RX (PD2), 115200波特率  
**用途**: TFmini Plus前向测距，用于避障

### 5. CAN总线接口
**连接器**: HDR-M_2.54_1x4P  
**标注**: "CAN1"  
**引脚**: 5V, GND, PD0, PD1  
**配置**: CAN1 (PD0=RX, PD1=TX)  
**用途**: 预留CAN总线扩展

### 6. 光流传感器接口
**连接器**: GH1.25-6P (6针座)  
**标注**: "光流"  
**引脚**: 3V3, GND, PC5, (空), PB5(R170-33Ω), PB4(R171-33Ω), PB3(R172-33Ω)  
**关键发现**: 
- PC5直接连接（可能是CS片选）
- PB3/4/5通过33Ω电阻连接
- 原理图右侧显示SPI1接口：PA4, PA5, PA6, PA7

**⚠️ 重要修正**: 
根据原理图右侧的"SPI1"标注（HDR-M_2.54_1x4P），光流传感器实际使用的是：
- **SPI1**: PA4(SCK), PA5(MISO), PA6(MOSI)
- **CS**: PC5 (片选)
- **PB3/4/5**: 可能是备用接口或其他功能

### 7. 数传接口
**连接器**: XH2.54-4P  
**标注**: "LORA"  
**引脚**: 5V, GND, PE8, PE7  
**配置**: USART8 (PE8/PE7), 115200波特率  
**用途**: 数传 433MHz数传模块

### 8. 空闲/预留接口
**连接器**: XH2.54-4P  
**标注**: "空闲/预留"  
**引脚**: 3V3, GND, PF5, PE0  
**配置**: USART8 (PE1=TX, PE0=RX)  
**注意**: 原理图显示PF5，可能是使能或复位引脚

### 9. IBUS接收机接口
**连接器**: XH2.54-4P  
**标注**: "IBUS"  
**引脚**: 5V, GND, PA3, (空)  
**配置**: USART2_RX (PA3), 115200波特率  
**用途**: FlySky IA6B接收机，14通道IBUS协议

### 10. PWM输出接口
**连接器**: SH1.0-6P (6针座)  
**标注**: "PWM"  
**引脚**: GND, PA8(R175-100Ω), PA9(R176-100Ω), PA10(R177-100Ω), PA11(R178-100Ω), (空)  
**配置**: TIM1_CH1-4, 50Hz PWM  
**特性**: 每路带100Ω下拉电阻，防止上电时ESC误触发

### 11. GPS接口
**连接器**: GH1.25-6P  
**标注**: "GPS"  
**引脚**: 5V, GND, PA13, PA14, PA13, PA14  
**配置**: 
- USART4 (PC6/PC7), 115200波特率 - GPS数据
- I2C3 (PA14=SCL, PA13=SDA), 400kHz - QMC5883L 罗盘
**用途**: GPS模块 + 集成罗盘，NMEA协议 + I2C罗盘数据

### 12. DEBUG串口接口
**连接器**: HDR-M_2.54_1x4P  
**标注**: "DEBUG"  
**引脚**: 5V, GND, PB6, PB7  
**配置**: USART1 (PB6=TX, PB7=RX), 115200波特率  
**用途**: 调试输出

### 13. SWD烧录接口
**连接器**: HDR-M_2.54_1x4P  
**标注**: "SWD"  
**引脚**: 3V3, GND, PB9, PB8  
**配置**: PB8=SWCLK/USB2DP, PB9=SWDIO/USB2DM  
**注意**: PB8/PB9也可用作I2C1或USB2

### PA13/PA14 最终用途：I2C3 罗盘
**连接器**: HDR-M_2.54_1x4P  
**标注**: "I2C3"  
**引脚**: GND, 3V3, PA14, PA13  
**配置**: I2C3 (PA14=SCL, PA13=SDA)  
**⚠️ 冲突**: PA13/PA14也是SWD接口，使用I2C3时无法使用SWD调试

### 15. 电源接口
**连接器**: HDR-M_2.54_1x4P  
**标注**: "POWER"  
**引脚**: 3V3, 3V3, 5V, 5V  
**用途**: 电源输出，供外部模块使用

## 二、板载外设分析

### 1. SD卡接口
**卡座**: XKTF-015-G (MicroSD卡座)  
**引脚**: 
- PB12: DAT2/CS
- PB15: CD/DAT3 (卡检测)
- PB13: CMD
- PB14: DAT0
**配置**: SPI2模式 (PB12=CS, PB13=SCK, PB14=MISO, PB15=MOSI)  
**上拉电阻**: R157, R159, R169 (10kΩ)

### 2. 蜂鸣器驱动
**电路**: PB0 → R181(1kΩ) → Q1(S8050 NPN) → 蜂鸣器(U16)  
**配置**: GPIO输出，高电平驱动  
**特性**: 使用三极管放大，可驱动5V有源蜂鸣器

### 3. LED指示灯
**电路**: PB1 → R131(1kΩ) → LED1(NCD0603Y2)  
**配置**: GPIO输出，低电平点亮  
**颜色**: 红色LED

### 4. 电池电压检测
**电路**: VBAT → R179(10kΩ) → PA1 → R180(2kΩ) → GND  
**滤波**: C105(100nF)  
**分压比**: 2kΩ/(10kΩ+2kΩ) = 1/6  
**计算公式**: 
```c
float vbat_voltage = (adc_value * 3.3 / 4096.0) * 6.0;
```
**测量范围**: 0-19.8V (4S满电16.8V → 2.8V ADC输入)

## 三、关键硬件修正总结

### 修正1: 光流传感器SPI接口（已确认）
**实际配置**: **SPI3** (PB3/4/5) + CS(PC5)  
**特性**: PB3/4/5通过33Ω电阻(R170/R171/R172)连接，用于阻抗匹配和信号完整性  
**连接器**: GH1.25-6P

### 修正2: SPI1预留接口
**配置**: PA4/5/6/7  
**连接器**: HDR-M_2.54_1x4P  
**用途**: 预留扩展接口，可用于额外的SPI传感器

### 修正3: SWD烧录接口（已确认）
**实际使用**: **PB8/PB9** 作为SWD烧录接口  
**PB8**: SWCLK / I2C1_SCL / USB2DP  
**PB9**: SWDIO / I2C1_SDA / USB2DM  
**连接器**: HDR-M_2.54_1x4P (标注"SWD")  
**复用说明**: PB8/PB9具备I2C1复用功能，但本项目最终配置中I2C1未使用；GPS 串口改用USART4 GPS (PC6/PC7)，SWD烧录时无需断开旧OLED/罗盘方案。

### 修正4: I2C接口分配（最终确认）
**USART4 GPS (PC6/PC7)** - 共享总线:
- 设备1: OLED OLED旧方案 (地址0x3C)
- 设备2: GPS (地址0x0D)
- PA14: I2C3_SCL
- PA13: I2C3_SDA
- 连接器: HDR-M_2.54_1x4P (OLED) + GH1.25-6P (GPS)
- 优势: 不与SWD冲突，可在线调试

**SWD接口 (PB8/PB9)** - 专用:
- PB8: SWCLK
- PB9: SWDIO
- 连接器: HDR-M_2.54_1x4P
- 独立接口，不影响I2C设备

### 修正5: 空闲/预留接口
**原理图显示**: 3V3, GND, PF5, PE0  
**USART8配置**: PE1=TX, PE0=RX  
**PF5用途**: 可能是空闲/预留的EN使能引脚或RST复位引脚  
**建议**: 在驱动中添加PF5的控制代码

## 五、代码修正建议

### 1. 光流驱动修正 (sensor_flow.c)

```c
// 错误配置
#define FLOW_SPI        SPI3
#define FLOW_SCK_GPIO   GPIOB
#define FLOW_SCK_PIN    GPIO_Pin_3

// 正确配置
#define FLOW_SPI        SPI1
#define FLOW_SCK_GPIO   GPIOA
#define FLOW_SCK_PIN    GPIO_Pin_4   // PA4
#define FLOW_MISO_PIN   GPIO_Pin_5   // PA5
#define FLOW_MOSI_PIN   GPIO_Pin_6   // PA6
#define FLOW_CS_GPIO    GPIOC
#define FLOW_CS_PIN     GPIO_Pin_5   // PC5

void sensor_flow_init(void) {
    // 使能SPI1时钟
    RCC_HB2PeriphClockCmd(RCC_HB2Periph_GPIOA | RCC_HB2Periph_GPIOC, ENABLE);
    RCC_HB1PeriphClockCmd(RCC_HB1Periph_SPI1, ENABLE);
    
    // 配置SPI1引脚
    GPIO_InitTypeDef GPIO_InitStructure;
    GPIO_InitStructure.GPIO_Pin = FLOW_SCK_PIN | FLOW_MISO_PIN | FLOW_MOSI_PIN;
    GPIO_InitStructure.GPIO_Mode = GPIO_Mode_AF_PP;
    GPIO_InitStructure.GPIO_Speed = GPIO_Speed_Very_High;
    GPIO_Init(FLOW_SCK_GPIO, &GPIO_InitStructure);
    
    // 配置CS引脚
    GPIO_InitStructure.GPIO_Pin = FLOW_CS_PIN;
    GPIO_InitStructure.GPIO_Mode = GPIO_Mode_Out_PP;
    GPIO_Init(FLOW_CS_GPIO, &GPIO_InitStructure);
    GPIO_SetBits(FLOW_CS_GPIO, FLOW_CS_PIN);  // CS默认高电平
    
    // SPI1配置
    SPI_InitTypeDef SPI_InitStructure;
    SPI_InitStructure.SPI_Direction = SPI_Direction_2Lines_FullDuplex;
    SPI_InitStructure.SPI_Mode = SPI_Mode_Master;
    SPI_InitStructure.SPI_DataSize = SPI_DataSize_8b;
    SPI_InitStructure.SPI_CPOL = SPI_CPOL_High;
    SPI_InitStructure.SPI_CPHA = SPI_CPHA_2Edge;
    SPI_InitStructure.SPI_NSS = SPI_NSS_Soft;
    SPI_InitStructure.SPI_BaudRatePrescaler = SPI_BaudRatePrescaler_64;  // 2MHz
    SPI_InitStructure.SPI_FirstBit = SPI_FirstBit_MSB;
    SPI_Init(FLOW_SPI, &SPI_InitStructure);
    SPI_Cmd(FLOW_SPI, ENABLE);
}
```

### 2. 电池电压ADC修正

```c
#define VBAT_ADC_CHANNEL    ADC_Channel_1   // PA1
#define VBAT_DIVIDER_RATIO  6.0f
#define ADC_VREF            3.3f
#define ADC_RESOLUTION      4096.0f

float get_battery_voltage(void) {
    uint16_t adc_raw = ADC_GetConversionValue(ADC1);
    float vbat = (adc_raw * ADC_VREF / ADC_RESOLUTION) * VBAT_DIVIDER_RATIO;
    return vbat;
}

// 电池电压告警
void check_battery_voltage(float vbat) {
    if (vbat < 14.0f) {
        // 4S低电，强制RTL
        trigger_rtl();
    } else if (vbat < 14.8f) {
        // 4S低电预警
        led_blink_fast();
        mavlink_send_statustext(MAV_SEVERITY_WARNING, "Low battery!");
    }
}
```
